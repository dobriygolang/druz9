# Docs Index

`docs/` — единая точка входа для внутренних markdown-файлов проекта.

## Правила

- Архитектурные карты и навигация по коду хранятся в `docs/maps/`.
- Feature-заметки и краткие сводки хранятся в `docs/features/`.
- Операционные инструкции и runbooks можно добавлять в `docs/runbooks/`.
- Локальные `README.md` рядом с кодом остаются допустимыми только если описывают конкретную папку или внешний vendor-пакет.

## Сейчас здесь

- `docs/maps/codebase.md` — карта ключевых модулей и где искать backend/frontend части.
- `docs/maps/backend-architecture.md` — целевая схема backend-модулей и правила разбиения.
- `docs/features/interview-prep.md` — текущая сводка по interview prep.
